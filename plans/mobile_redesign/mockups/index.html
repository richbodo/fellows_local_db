Mobile Redesign — Mockups

Tokens

  • Body / display: system-ui, -apple-system, ... — same stack as today
  • Mono: ui-monospace, "Cascadia Code", Menlo, ...
  • Page bg: #f5f5f8 (current body background)
  • Brand purple: #4a2c6a — primary action, focus ring
  • Link blue: #0066cc — high contrast for fellow names
  • Warning yellow: #fff3cd / #ffe69c / #664d03 family
  • Danger red: #7a1f1f / #fde2e2 family
  • Phone breakpoint: ≤480px — focus-mode by default
  • Tablet breakpoint: ≤768px — same rules, looser gutters
  • Touch targets: 44pt minimum on every interactive control
  • Inputs: 16px text — avoids iOS auto-zoom on focus
  • Safe areas: top + bottom insets respected (iOS notch / home indicator)

Reviewing on a phone

Each route file collapses its phone-frame chrome at viewports ≤480px, filling the screen the way the design would once ported. To preview on your actual Android run just serve-lan to start the dev server with the LAN IP printed, then open http://<ip>:8765/plans/mobile_redesign/mockups/index.html from the phone. The dev server already binds to all interfaces.
