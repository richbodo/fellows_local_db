# The PWA Bulletproofing & Distribution Guide

This document outlines best practices for ensuring a Progressive Web App
(PWA) is universally installable across Android (Chrome/Samsung), iOS
(Safari), and desktop environments.

## 1. The Manifest: The Source of Truth

The `manifest.json` is the first thing a browser checks. If it isn't
perfect, the "Install" button may never appear.

- **Provide maskable icons.** Modern Android (especially Samsung's One UI)
  uses "squircles." If you don't provide a maskable icon, the installation
  might fail or look broken.
  - Best practice: include a 512×512 px icon with `"purpose": "any maskable"`.
- **Align `start_url` and `scope`.** Ensure your scope covers your
  `start_url`. If they don't match, the browser may treat the app as a
  simple bookmark rather than a standalone app.
- **Avoid native-app conflicts.** If your manifest contains
  `related_applications`, ensure the IDs are correct. A mismatched ID
  can trigger the "Older Version" error on Android as it tries to find a
  non-existent Play Store app.

## 2. Solving the "Older Version of Android" Error

This error typically occurs when the WebAPK (the mini-Android package
generated by Google/Samsung servers) fails to "mint."

- **The SDK conflict.** If you are distributing via an APK (Trusted Web
  Activity), ensure your `targetSdkVersion` is at least 34 (as of
  2025/2026).
- **The "minting" failure.** If installing directly from the browser,
  this error is often a generic fallback for a failed background process.
  - **Fix:** check your `share_target` in the manifest. If it uses
    `method: "POST"`, try switching to `GET`. Some WebAPK servers reject
    POST-based PWAs for security reasons.
- **Home-screen locks.** On Samsung devices, if "Lock Home Screen Layout"
  is enabled in System Settings, the installation will fail with a
  cryptic error.

## 3. iOS (Safari) Optimization

iOS does not support the `beforeinstallprompt` API, meaning you cannot
trigger an "Install" button via code.

- **Manual instruction.** Use a custom UI component to detect iOS and
  show a "How to install" tooltip.
- **The Safari rule.** Remind users that while Chrome on iOS can install
  PWAs (iOS 16.4+), Safari remains the most stable environment for
  persistent local storage (IndexedDB).
- **Standalone mode.** Use the following logic to show/hide instructions
  only when needed:

```js
const isStandalone =
  window.matchMedia('(display-mode: standalone)').matches ||
  window.navigator.standalone;
```

## 4. The Service Worker "Heartbeat"

A PWA is not "installable" unless it has a functional service worker.

- **The `fetch` event.** You must have a `fetch` event listener, even if
  it's a pass-through. Without it, Chrome's installability criteria are
  not met.
- **Update logic.** If you update your app, the service worker may keep
  an old version in the user's cache. Implement a "New version
  available" refresh toast to prevent "it works on my machine" support
  tickets.

## 5. Diagnostic Tools for Users

When a user says "it won't download," point them to a diagnostic tool
instead of asking for their system settings.

| Tool                   | Purpose                                                       |
| ---------------------- | ------------------------------------------------------------- |
| What PWA Can Do Today  | Checks hardware/API support (Bluetooth, DB, etc.).            |
| PWAHero                | Custom install guide based on the user's specific browser.    |
| Progressier Diagnostic | Clean "green/red" interface for non-technical users.          |

## Similar Labors & Industry Standards

You are not the first to fight the "installability" war. Here are the
projects that have paved the way:

- **Microsoft PWA Builder.** The gold standard for validating manifests
  and packaging PWAs for stores. Their "Report Card" feature is
  essential for debugging.
- **Google Web.dev (PWA).** The official documentation for
  "installability criteria."
- **The PWA Install Component.** A community effort to create a
  one-size-fits-all install button that handles the logic for iOS,
  Android, and desktop automatically.

## Pro-Tip for Your Project

Since your users are finding it hard to follow instructions, consider
adding the `pwa-install` web component to your repository. It replaces
"go to your settings" with a single, smart button that changes its
instructions based on whether the user is on a Samsung, an iPhone, or a
PC.
